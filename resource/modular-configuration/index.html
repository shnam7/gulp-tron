Modular Configuration

gulp-tron is designed with managing mulple sub-projects in mind. Typically, main gulpfile located in project root loads each gulpfiles from sub-projects. Each sub project, typically located in its own sub-directory, can have its own gulpfile using gulp-tron.

Quick Example#

// file: 'docs/gulpfile.js
const tron = require('gulp-tron');
// make some BuildConfig's
// ...
module.exports = tron.createProject(/*...*/);
// file: 'example/gulpfile.js
const tron = require('gulp-tron');
// make some BuildConfig's
// ...
module.exports = tron.createProject(/*...*/);
// file: 'gulpfile.js
const tron = require('gulp-tron');
const docs = require('./docs/gulpfile.js'); // docs is an instance of gulp-tron project
const example = require('./example/gulp.js'); // example is an instance of gulp-tron project

Check samples for more examples.
