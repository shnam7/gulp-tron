gulp-tron

gulp-tron API is available immediately once it is loaded.

const tron = require('gulp-tron');

API#

createProject()#

createProject(buildItems: BuildItem | BuildItems = {}, opts?: ProjectOptions) => GProject;

Create project instance and returns it. The projet is added to the project iist of gulp-tron.

Parameters#

parametertypedescription
buildItemsBuildItem or BuildItemsSingle build item or Object of multiple build items. See below Builditem
optionsProjectOptionsSee belo ProjectOptions

BuildItem#

BuildItem is one of following 3 configuration types. Refer to Builders section for more details.

  • BuildConfig
  • WatcherConfig
  • CleanerConfig

BuildItems is an object with multiple BuildItem configurations.

const buildItem1 = { name: 'scss', builder: 'GCSSBuilder', /*...*/}
const buildItem2 = { name: 'scripts', builder: 'GTypeScriptBuilder', /*...*/}
const cleaner = { builder: 'cleaner' }
const watcher = { builder: 'watcher' }
const buildItems = { buildItem1, buildItem2, cleaner, watcher }

ProjectOptions#

parametertypedefaultdescription
projectNamestringundefinedOptional project name
prefixstring""Optional project prefix. This is prefixed to names of all the build items in the project when they are resolved.
customBuilderDirsstrinf or strinf[]undefinedDirectory list to search for builder class names

getBuildNames()#

getBuildNames(selector: string | string[] | RegExp | RegExp[]): string[]

Returns an array of build names from all the projects that matches the pattern in selector.

findProject()#

findProject(projectName: string) => GProject | undefined;

Returns first project that matches parojectName. projectName can be optionally assigned when it's created by createProject() function. If not found, returns undefined.

setPackageManager()#

setPackageManager(packageManager: string | PackageManagerOptions) => void;

Sets package manager options. packageManager can be string value of 'npm', 'pnpm' or 'yarn'. In this case, package install option is set to default which is installing the module as devDependencies. If packageManager is object type, then it can have following propeties.

Propertytypedefaultdescription
namestringundefinedName of the package maager
installComandstringundefinedPackage installation command (ex: npm i)
installOptionsstringundefinedpackage installation options (ex: --save-dev)
autoInstallbooleanundefinedEnable automatic module installation

Undefined properties are simply ignored. Refer to auto-install section for details.

series()#

series(...args: BuildSet[]) => BuildSetSeries;

Convert argument list into series type build set.

parallel()#

parallel(...args: BuildSet[]) => BuildSetParallel;

Convert argument list into parallel type build set

registerExtension()#

registerExtension(name: string, ext: RTBExtension) => void;

Register RTB extension under the given name. Refer to Extension for details.

loadExtension()#

loadExtension(globModules: string | string[]) => void;

Load extension modules from the files specified by globModules. globModules can be single glob expression or array of glob expressions for the extension files. Refer to Extension for details.

require()#

require(id: string) => any;

Load module identified id. This is the same as node require() function except that it check module automatic Installation option. If automatic installation is enabled and the module(id) is not installed, then it is installed before it is loaded. Refer to auto-install for details.

Properties#

conf#

Global RTB list which contains all the RTB instances created.

rtbs#

Global RTB list which contains all the RTB instances created.

builders#

Object containing all the built-in builder classes, which can be used in custom builder creation.

Usage#

const tron = require('gulp-tron');
class MyBuilder extends tron.builders.GBuilder {
// ...
}
